Windows

VPN Connection (Reccomended)

The easiest method to connect is through the Universities official VPN. The reccomended client to use is Pulsesecure

Download (64 bit): Desktop app
Download (32 bit): Desktop app (32 bit)

Legacy downloads (Windows 8 & prior)
Download (64 bit): Legacy Installer
Download (32 bit): Legacy Installer (32 bit)

Once pulse secure is downloaded and installed with the reccomended settings, click the + button and add the following connection.

Type: "UAC" or "SSL-VPN"
Name: York university
Server URL: https://webvpn.york.ac.uk/

Finally click connect, enter your University username and password and Vwulla. You're done! You now have a secure connection.


SSH Connection (Alternative Method)

If you are un-able to establish a VPN connection you can instead use SSH to connect. SSH requires a bit more to setup but is an equally viable alternative.

Note that when connecting to servers through ssh you should use a local (lan) style connection instead. For Minecraft this is equivalent to using an i.p address of localhost

Make sure you have SSH access
To use SSH to connect you must first have SSH access through the university. To request access email: itsupport@york.ac.uk stating that you would like to gain SSH access. For more info visit: link

Enable SSH through Windows
SSH is not enabled on Windows by default. To enable it start 'Settings' then go to Apps > Apps and Features > Optional Features and choose to add a feature, then select 'openSSH Server & Client'

Download and run the .bat file provided
For simplicity we have provided a file you can download and run immediately without any additional setup to establish a secure connection to our gameservers. None of the data inputted is stored or collected. You may recieve a security warning when running the file. Please ignore this, thank you.

Download: Secure Download (Google Drive)

Common SSH issues

  • It asks for my userid and then crashes > You need to make sure you have openSSH Client & Server installed on your system.
  • The program is saying 'Permission denied, please try again' > The most likely scenario is that ssh hasn't been enabled for your account. Email itsupport to gain access to ssh through the university.
  • After I type my password the program is left hanging (nothing else happens) > This is what we expect to happen, for some reason windows ssh dosen't prompt you to let you know that the info you entered is correct. You are definately connected though if this happens.
  • When I type my password it dosen't show up > This is standard practice for terminal/commandprompt. The password is being entered but it wont show the password being entered for security reasons.
